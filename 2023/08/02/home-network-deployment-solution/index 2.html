家庭网络方案（理论篇）

发表于 2023-08-02 Valine：

背景

最近要开始新房子的装修了，正好对目前的家里使用的网络方案也不是很满意，借着这个机会，整理一下网络的方案，也会分开说说不同场景选型的一些方案。

前几年做家庭方案的时候，基本上上限就是千兆网络，在很长一段时间，全屋无线+有线全千兆网络是家装网络的天花板。但在2023年的今天，如果你想要打造一套战未来的基础网络设施，我会推荐你考虑2.5G的网络环境，甚至10G，也就是万兆网络。

本文介绍家庭网络的理论方案，包括无线方案、永久链路的选择等等方面介绍。

方案选择

无线方案

无线方案的选择，主要为户型较大、户型较复杂（如狭长型、承重墙较多等）、楼层较多、网络要求较高等场景提供的，如果户型不大且对边角网络要求不高，只需要买一台功率较大的路由器放在房子中心位置，那么一般来说，在边角位置也能搜索到2.4G网络，对于这类场景，可以跳过此章节。

有些人会认为，需要什么无线方案，只要多买几个路由器放在家里的不同位置，用同一个WiFi名称（SSID）和相同无线密码，不就是可以实现全屋WiFi覆盖了吗？这种方案的确可以实现全屋覆盖的诉求，但是体验一定是不够好的。因为这样部署，如果你的移动设备连着A路由器走到B路由器的旁边，并且在此处还可以搜到A的信号的话，移动设备并不会自动断开A的连接去连接B，哪怕A的信号已经非常微弱了。因为你的移动设备是不知道此处还有一个更适合的B路由器在提供服务，这就会出现你明明站在路由器边上，仍然上不了网的情况。遇到这种情况，往往需要手动去断开WiFi，然后等移动设备自动连上信号更强的那台设备。

对路由器略有研究的同学可能会想到，路由器里是有一个设置可以实现自动剔除弱信号设备，设置一个信号阈值，如果信号比阈值弱，将会自动把设备踢下线，那么移动设备就可以再次连接信号更强的路由器了。这个方案比上一个的确更有，降低了需要手动切换网络的概率，但是有两个比较明显的问题：

  1. 剔除阈值比较难定。因为家庭环境比较复杂，有墙体或者其他的阻隔，很难保证在剔除以后立刻有一个更适合的路由器能接管，例如一些边缘位置，不大可能再去布置一台路由器来补充信号覆盖，本来信号弱一点至少能用，剔除了以后就彻底不能用了。
  2. 无法实现无缝衔接。因为这种形式，本质上还是断开上一个路由器，搜索下一个并且连接，那势必会导致网络的抖动，你想象一下你正在视频通话或者正在手游中厮杀，网络要断几秒几十秒不等的感受。

铺垫够多了，直接说一下解决方案：WiFi漫游，关于什么是WiFi漫游和为什么需要，直接引用华为官网的介绍

什么是WiFi漫游？

WiFi漫游是指无线终端STA在同属一个ESS（Extended Service Set）的不同AP的覆盖范围之间移动且保持用户业务不中断的行为。
WLAN网络的最大优势就是STA不受物理介质的影响，可以在WLAN覆盖范围内四处移动并且能够保持业务不中断。某一区域内多个AP使用相同的SSID（Service Set Identifier），那么在该区域内用户看到的无线网络标识是同一个。当STA从一个AP覆盖区域移动到另外一个AP覆盖区域时，利用WiFi漫游技术可以实现STA用户业务的平滑切换。

为什么要有WiFi漫游？

一台AP的无线信号只能覆盖有限的区域范围。想象一下，如果没有WiFi漫游，那么当我们在WLAN网络中移动到AP信号弱的区域时，我们的无线终端（下文简称为STA，例如手机、平板、笔记本电脑等）与AP之间的网络将会断开。对于语音、视频等业务，网络断开将严重影响我们的使用体验。

而WiFi漫游的出现正是为了解决这一问题。如下图所示，AP1和AP2提供相同的接入网络服务（SSID都是huawei-net）。STA移动到两台AP覆盖范围的临界区域时，STA与新的AP2进行关联并与原有AP1断开关联，且在此过程中保持不间断的网络连接。简单来说，就如同手机的移动通话功能，手机从一个基站的覆盖范围移动到另一个基站的覆盖范围时，能提供不间断、无缝的通话能力。


WiFi漫游

WLAN网络的最大优势就是STA不受物理介质的影响，可以在WLAN覆盖范围内四处移动并且能够保持业务不中断。WiFi漫游提升了用户使用无线网络的体验，解决了以下问题：

  • 保证用户IP地址不变，漫游后仍能访问初次上线时关联的网络，且所能执行的业务保持不变。
  • 避免漫游过程中用户的认证时间过长而导致数据丢包甚至业务中断。

废话不多说了，目前家用实现WiFi漫游的主要是两种方案：Mesh 组网和 AC+AP

Mesh 组网

Mesh组网是指多个路由器（一般情况是同平台的）直接，厂家开发了一套通讯协议，可以将若干路由器划分为主子路由器，所有子路由器接入主路由网络，将子路由摆放到家里不同的位置，来实现全屋WiFi覆盖。而主路由将可以协调各子路由（包括它自己）跟移动设备，保证移动设备一直连接在最佳路由器下。如果设备在移动，需要切换连接的路由器，也可以实现无缝切换，不影响设备的正常使用。

Mesh组网有个最大的优点是可以实现无线Mesh，即子路由与主路由之间的连接也可以是无线的，不需要通过网线连接，这种方式叫无线回程。听起来很美好是不是，但是无线回程将会损失一半的无线带宽，例如厂商宣称5G频段可以实现3000Mbps速率，如果使用了无线回程，那么5G频段下理论最大速率就只有1500Mbps了，如果非要使用这种方式，建议选择有两个5G频段的路由器，那么你仍可以使用一个满血的5G频段。

Mesh还有一个优点就是成本较低，此类路由器就是普通路由器，你可以先买一个回来使用，如果觉得信号不够好，再买若干个补充即可，以红米AX3000举例，两百多一个，一般家庭放三个基本可以实现全覆盖。如果对速率有更高要求，AX6000，AX9000也都可以考虑的。

Mesh的主要是如下几点：

  1. 子设备数量一般不多，例如小米限制最多9个子路由，大户型会不够用
  2. 路由器体积比较大，一般也说不上好看，可能跟你的家装风格冲突
  3. 需要额外供电，没有办法直接通过网线供电，要多拖一根小尾巴（至少目前我还没有见过poe供电的子路由）

AC + AP

AC + AP 跟mesh中的主子路由有点类似，有一个AC控制若干个AP来实现WiFi漫游。而AC控制器有时候会跟路由器集成在同一台设备中，采用此方式组网，使用

要说ACAP的优点，描述再多不如说他的应用场景：企业无线的唯一解决方案。企业级解决方案说明这个方案是能兼顾稳定性和可扩展性的，不过有些时候企业级也不是优点，例如华为的AirEngine方案，普通的用户根本玩不明白，是不适合家用的。

ACAP适合对网络稳定性要求的场景，并不是说所有的ACAP都比mesh稳定，只是说ACAP的上限比mesh更高；而且mesh对子路由的数量是有限制的，如果你的家里足够大，那么也只能使用ACAP方案

选择建议

  1. 如果户型简单，面积不大，对网络要求也一般，那么在中心位置放一台性能强劲的路由器即可
  2. 除1的情况外，如果是一般的平层或不算太大的跃层，并且也不想太折腾的，个人觉得mesh组网已经足够，除非你觉得家里摆各种奇形怪状的路由器不好看
  3. 如果你的家里足够大，尤其是有很多层，甚至希望在花园中也有满格网络覆盖，不要挣扎了，用ACAP吧

永久链路

内网带宽

关于带宽，其实开头已经讲了，2023年我建议对网络有追求的同学，2.5G起步，土豪可以上10G。有人会说，现在外网最多也就1000Mbps，内网这么快有什么用？且不说很多人家里会考虑NAS、影音系统等，光说运营商带宽，2000Mbps已经在小范围运行了（目前咨询杭州电信和联通都用FTTR 2000M的服务了），到时候内网千兆将是你家的瓶颈。

对于爱折腾的玩家来说，内网带宽更是能让你如鱼得水的基础建设，你的所有文件都可以存储在NAS中，其他设备本机只需要很小的磁盘空间，因为随时都能快速访问数TB的NAS，高清的蓝光电影也可以直接在电视上播放。

所以我本次装修的方案，将会以2.5G为主，并保留后续升级10G甚至更大带宽的可能性。

布线

很多同学在新房装修的时候，大概听说过网络分为多少类的，觉得数字越大越好，直接上6类甚至7类的；也包括我上一套精装修房子出售的时候，买家会关心开发商用的是几类线。此处直接说结论，50m以内普通质量合格的五类网线基本能跑到万兆。

所以如果你是精装的房子，一般情况下，我不会建议你去更换网线，除非你对网络波动特别敏感，或者准备上10G以上的内网。

但如果是新装修的，我会建议你使用6类普通网线+多芯单模光纤。

先说网线，6类网线家用足以，如果你对稳定性有要求，可以上超六类带屏蔽层的，如果上了超六，务必确认施工师傅的专业性，要用超六的水晶头、做好接地、用正确的方式打水晶头/网络模块。最后一条很重要，因为购买并使用对应水晶头很容易，接地的话选三插的交换机也不难，但是正确的方式打水晶头/网络模块并不是每个师傅都会的，因为这类网线的屏蔽层（有些还会有地线）需要正确安装才能起到屏蔽的作用。

光纤的作用一般是战未来的，尤其是光纤非常便宜，所以预埋上也花不了多少钱，尤其是一根N芯光纤相当于N根光纤的集合，拥有它，将拥有无限未来。光纤还有一个作用是连接多个交换机的，对于多层用户会很有用，这个后面会具体展开说。
